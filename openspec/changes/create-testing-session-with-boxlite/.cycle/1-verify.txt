**WARNINGS (1):**

1. ⚠️ **POST /connect endpoint has no e2e test coverage.** The `POST /connect` endpoint is implemented in `kawa.ts:61-79` with three scenarios (200 success, 400 missing link, 503 not ready), but no test exercises it. Design decision D3.1 explicitly specifies this endpoint, and the `e2e-alice-testing` spec has three scenarios for it. The test harness connects Alice via `apiConnectActiveUser()` directly through the SDK, bypassing the HTTP endpoint entirely.
   - **Recommendation:** Add a unit-level or integration test for `POST /connect`, or add an e2e scenario where a second peer connects to Kawa via `fetch('http://localhost:18080/connect', { method: 'POST', body: link })`.

**SUGGESTIONS (2):**

1. 💡 **Task numbering gap:** Section 4 tasks skip from 4.1 to 4.3 (no task 4.2). This is cosmetic only — no task is missing.

2. 💡 **Per-suite vs per-run setup:** Each of the 6 test suites independently calls `setupShared()` / `teardownShared()`, meaning Kawa and Alice processes are started and stopped 6 times during a full `npm run smoke` run. This is correct per the spec ("create and destroy test environment per suite run"), but a shared-process approach (one setup/teardown for all suites) would reduce total test time from ~15–25 min to ~3–5 min. This is a potential optimization for a future iteration, not a correctness issue.
